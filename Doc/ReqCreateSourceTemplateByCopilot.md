新規機能追加のために Copilotに渡す情報は何が櫃世でしょうか？
---
新規機能追加のために Copilot に渡すべき情報を整理します。現在お持ちの情報に基づいて、**不足している重要な情報**を以下にまとめます。

## 📋 既に揃っている情報
✅ リポジトリ構成（Doxygen設定で確認可能）
✅ 言語構成（C 97.4%, C++ 1.3%, Assembly 1.3%）
✅ プロジェクト名（BluePill STM32F103C8基板プロジェクト）
✅ ドキュメント構造（Doc/Doxyfile で確認可能）

## 🔍 **追加で必要な情報**

### 1. **プロジェクト構造とファイル配置**
```
- ソースコードの主要ディレクトリ構成
- Src/, Inc/, HAL/, FreeRTOS/ などの役割
- 主要なファイル一覧
```

### 2. **現在の機能と仕様**
```
- STM32F103C8 の使用ペリフェラル（UART、SPI、ADC など）
- 主要な機能モジュール
- 現在実装されている機能リスト
```

### 3. **追加したい新規機能の詳細**
```
- 機能概要
- 要件・仕様
- 実装対象ペリフェラル
- 既存コードとの連携方法
```

### 4. **コーディング規約・ガイドライン**
```
- ネーミング規約
- ファイル構成ルール
- コメント規約
```

### 5. **ビルド・テスト情報**
```
- ビルドツール（CMake、Make など）
- コンパイラ設定
- テスト環境
```

---

## 💡 **次のステップ**

新規機能を追加する際は、以下の情報を Copilot に提供してください：

```markdown
1. 追加したい機能の説明
2. ファイルの現在の構成図
3. 参考にすべき既存コード（類似機能がある場合）
4. プロジェクトの命名規約やコード標準
5. 具体的な実装要件
```

**リポジトリを確認して、これらの情報を提供いただければ、より正確な支援が可能です。**
